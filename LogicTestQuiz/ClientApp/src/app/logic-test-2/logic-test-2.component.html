Answer 2 :
  Interface คือ Class คือ ที่มีเฉพาะตัว Method header เท่านั้น ไม่มีการ implement การทำงานอะไร โดยการใช้งานของ Interfaces นั้นจะนำไป Implement โดยคลาส สำหรับการกำหนดการทำงานให้กับเมธอด
  Abstract เหมือนกับ Interface แต่จะมีข้อกำหนดที่ยืดหยุ่นกว่า เพราะสามารถที่่จะเพิ่ม Method และการทำงาน implement เข้าไปใน Class ที่เป็น abstract ได้
เมื่อไหรควรใช้ Interface หรือควรใช้ Abstract ?
  ใช้ Interface Class เมื่อต้องการวางกรอบ ขอบเขตคร่าวๆ เพื่อให้ subclass เอาทำ implement การทำงานภายในต่อได้เอง   ใช้ Abstract Class เมื่อรู้ว่ามีการทำงานบางอย่างที่ ซ้ำๆกันใน subclass โดยยกขึ้นมาเขียนใน Abstract Class เลย และยังเป็นการทำ template ป้องกัน fragment จากการที่ DEV ไป implement กันเอง เพราะต่างคนต่าง Logic แล้วภายหลังจะ maintain ลำบาก
